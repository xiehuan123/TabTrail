# TabTrail 本地安装说明

## 构建

```bash
npm run build
```

构建产物会生成在 `dist/` 目录。该目录可以作为未打包扩展加载到 Chromium 浏览器。

## Chrome 加载方式

1. 打开 `chrome://extensions/`。
2. 开启右上角“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择本项目的 `dist/` 目录。
5. 点击浏览器工具栏中的 TabTrail 图标测试 popup。
6. 在扩展详情中打开侧边栏入口，测试 side panel。
7. 打开新标签页，确认默认展示 TabTrail 工作台。

## Edge 加载方式

1. 打开 `edge://extensions/`。
2. 开启“开发人员模式”。
3. 点击“加载解压缩的扩展”。
4. 选择本项目的 `dist/` 目录。

## 新标签页与专注整理

TabTrail 会把浏览器普通新标签页替换为 TabTrail 工作台。这个页面用于搜索当前打开标签、查看最近活跃、查看最近关闭、整理分组和打开侧边栏。

TabTrail 不能直接隐藏浏览器原生顶部标签栏，这是 Chromium 扩展 API 的边界。专注整理模式提供的是替代方案：使用新标签页工作台、侧边栏、批量关闭和最近关闭恢复来减少顶部标签栏拥挤。

如果要恢复浏览器默认新标签页，需要禁用 TabTrail 扩展，或后续使用不包含新标签页覆盖的构建。

## 手动验证清单

- 打开新标签后，最近活动记录增加。
- 普通新标签页展示 TabTrail 工作台。
- 切换标签后，popup 最近活跃区域更新。
- 关闭标签后，popup 和 side panel 的“最近关闭”出现该标签。
- 从 popup 点击最近活跃标签，可以切回原标签。
- 从 popup 点击最近关闭标签，可以重新打开 URL。
- side panel 默认展示当前窗口，切到“全部窗口”后展示所有窗口标签。
- 搜索能按标题、URL、域名过滤。
- side panel 内上移/下移只改变预览顺序。
- 点击“应用到标签栏”后，真实标签栏顺序改变。
- 关闭 1-2 个标签不弹确认；关闭 3 个及以上弹确认。
- 专注整理说明明确提示不能直接隐藏浏览器原生顶部标签栏。
- TabTrail 固定只影响插件展示，不改变浏览器原生 pinned 状态。
