## 背景

TabTrail 是一个从零开始的浏览器插件项目。当前仓库只有 OpenSpec 结构，还没有扩展代码。第一版目标浏览器为 Chrome / Edge 等 Chromium 浏览器，采用 Manifest V3。

用户已经确认第一版重点覆盖两个痛点：找回最近操作过的标签，以及整理当前打开的大量标签。产品不做完整浏览历史、不读取网页正文、不把最近操作记录同步到云端。

## 目标 / 非目标

**目标：**

- 用后台 service worker 监听标签打开、激活、关闭等事件，维护最近 100 条本机活动记录。
- 用 popup 提供快速恢复入口：最近活跃 8 个、最近关闭 8 个。
- 用 side panel 提供完整整理入口：当前窗口 / 全部窗口切换、网站分组、手动分类、搜索、预览排序、批量关闭。
- 最近关闭作为系统分类存在，内容只由插件维护，允许一键清空。
- 分类、固定状态、排序偏好、视图偏好等轻量元数据通过 `chrome.storage.sync` 同步。
- 高频且更敏感的最近操作记录通过 `chrome.storage.local` 保存在本机。

**非目标：**

- 不支持 Firefox 作为第一版目标。
- 不同步最近操作记录。
- 不读取或分析网页正文内容。
- 不实现云端账号、服务端同步或 AI 分类。
- 不把手动分类做成书签系统；第一版手动分类只作用于当前打开标签。
- 不自动改变真实浏览器标签顺序；只有用户点击“应用到标签栏”后才同步。

## 关键决策

### 决策：使用 Chromium Manifest V3

选择 MV3 是因为 Chrome / Edge 是第一版目标，MV3 是 Chromium 扩展当前主流架构。后台逻辑放在 service worker，UI 分为 popup 和 side panel。

替代方案是同时支持 Firefox，但侧边栏、权限和扩展 API 差异会增加 MVP 风险。第一版先不承担跨浏览器兼容成本。

### 决策：弹窗和侧边栏分工

Popup 只承担快速找回，展示最近活跃 8 个和最近关闭 8 个，避免把小弹窗做成完整管理界面。

Side panel 承担整理工作，包括窗口范围切换、分组、搜索、拖拽预览排序、多选和批量关闭。这样符合用户对“快速入口”和“整理工作台”的不同使用场景。

### 决策：本地记录与同步设置分层

最近操作记录写入 `chrome.storage.local`，只保留最近 100 条。该记录变化频繁且更敏感，不进入同步存储。

用户偏好、手动分类名称、分类映射、插件内固定状态和排序偏好写入 `chrome.storage.sync`。这些数据体积小、变化低频，适合跨设备保持一致。

### 决策：最近关闭是只读系统分类

“最近关闭”不是普通手动分类。它由关闭事件自动维护，用户不能拖拽、改分类、单条删除、手动添加或移出。用户只能一键清空全部最近关闭记录。

这个设计避免最近关闭被误当成书签或长期资料库，同时仍然提供隐私控制。

### 决策：排序先预览，确认后应用

侧边栏内拖拽只改变 TabTrail 的预览顺序，不立即改变浏览器真实标签栏。用户点击“应用到标签栏”后，插件才调用标签移动能力同步顺序。

替代方案是拖拽即同步，但误操作成本较高。预览确认更适合第一版。

### 决策：插件内固定不等于浏览器原生 Pin Tab

TabTrail 的固定只影响插件内部排序和展示优先级，不调用浏览器原生固定标签能力。这样可以提升找回效率，同时避免改变用户真实浏览器状态。

## 风险 / 取舍

- 风险：MV3 service worker 生命周期可能导致内存状态丢失。缓解：关键状态必须落到 `chrome.storage.local` 或 `chrome.storage.sync`，UI 打开时从存储和当前标签状态重建。
- 风险：`chrome.storage.sync` 容量和写入频率有限。缓解：只同步低频轻量元数据，不同步最近活动记录。
- 风险：关闭标签后部分标签信息可能不完整。缓解：在关闭前通过标签事件和当前缓存保存标题、URL、域名、窗口信息和时间戳。
- 风险：批量关闭可能误操作。缓解：关闭 1-2 个直接执行，关闭 3 个及以上必须确认；关闭后进入最近关闭。
- 风险：Edge 侧边栏兼容细节可能与 Chrome 不完全一致。缓解：MVP 以 Chrome 行为为准，Edge 作为 Chromium 兼容目标进行验证。
- 风险：手动分类只作用于当前打开标签，可能被用户误解为书签。缓解：UI 文案和交互避免暗示长期保存，关闭标签后不在手动分类中保留。

## 迁移计划

这是新项目，没有既有数据迁移。

实施时先建立扩展骨架和存储模型，再接入标签事件，最后实现 popup 和 side panel。若某个 UI 入口出现问题，可以保留后台记录能力并单独回退对应入口。
