## 背景与动机

经常打开大量浏览器标签的用户，需要一种更快的方式回到刚刚使用过的标签，并整理当前窗口里的标签混乱。浏览器历史和书签不能直接回答“我刚才在哪个标签？”或“我现在开着这么多标签，怎么快速整理？”。

## 变更内容

- 新增一个基于 Chromium Manifest V3 的浏览器插件 MVP，名称为 TabTrail。
- 在本机记录最近标签操作：打开、切换、关闭。
- 提供插件弹窗，用于快速回到最近活跃标签和恢复最近关闭标签。
- 提供浏览器侧边栏，用于整理当前窗口或全部窗口的打开标签。
- 支持按网站域名自动分组，并支持用户为当前打开标签建立手动分类。
- 提供只读的“最近关闭”系统分类，该分类内容由插件维护，用户只能整体清空。
- 支持侧边栏内拖拽预览排序，并通过明确的“应用到标签栏”动作同步到真实浏览器标签栏。
- 支持在当前视图内按标题、URL、域名本地搜索打开标签。
- 支持多选标签并批量关闭；关闭 3 个及以上标签时必须确认。
- 支持插件内固定标签，用于提升 TabTrail 内展示优先级，但不改变浏览器原生固定标签状态。
- 最近操作记录仅保存在本机；分类、固定、排序偏好和其他轻量设置通过浏览器同步存储跨设备同步。

## 能力范围

### 新增能力

- `recent-tab-activity`: 记录并暴露最近打开、切换、关闭的标签活动，并执行本地保留数量限制。
- `quick-recovery-popup`: 提供浏览器插件弹窗，展示最近活跃标签和最近关闭标签，支持快速切换或重新打开。
- `tab-organization-panel`: 提供侧边栏，用于按窗口范围整理标签、分组、搜索、预览排序和批量操作。
- `tabtrail-preferences-sync`: 定义哪些 TabTrail 设置和元数据可跨设备同步，哪些数据必须仅保存在本机。

### 修改能力

无。

## 影响范围

- 新增 Chromium MV3 插件结构，包括 manifest、后台 service worker、弹窗 UI、侧边栏 UI 和存储模块。
- 使用浏览器扩展 API 管理标签、窗口、侧边栏和存储。
- 需要标签访问、侧边栏和扩展存储相关权限。
- MVP 不包含服务器、账号系统、外部上传浏览数据、读取网页正文或完整浏览历史追踪。
