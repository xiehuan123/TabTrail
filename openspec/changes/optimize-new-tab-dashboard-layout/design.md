## Context

TabTrail 已有 popup、side panel 和新标签页仪表盘。当前新标签页通过 `chrome_url_overrides.newtab` 接管浏览器新标签页，能展示最近活跃、最近关闭、标签分组和整理操作，但布局仍偏“管理后台三栏视图”。当用户打开大量标签时，分类列表纵向堆叠，扫描成本高；同时页面仍保留“打开侧边栏”按钮，弱化了新标签页作为默认整理入口的定位。

用户给出的参考方向是资讯聚合站式分类导航：顶部或主体区域先展示一排多个分类标题，用户先选分类，再浏览该分类内容。这个模式更适合高频扫读和大量标签整理。

## Goals / Non-Goals

**Goals:**

- 让新标签页仪表盘变成分类优先的默认整理入口。
- 分类标题区在桌面宽度下一行展示多个分类，支持自动换行，数量多时仍可扫读。
- 点击分类标题后，内容区只展示当前分类的标签。
- 支持在仪表盘中新建手动分类，并将选中标签归入分类。
- 支持把打开标签拖拽到分类标题上完成归类。
- 支持关闭普通分类中的所有当前打开标签，并在关闭前确认。
- 保护“最近关闭”系统分类：只读、不可拖入、不可被关闭分类操作处理。
- 从新标签页移除“打开侧边栏”入口。
- 最后升级插件版本并补充 README。

**Non-Goals:**

- 不把分类做成永久书签系统；手动分类仍只作用于当前打开标签。
- 不允许编辑或移动最近关闭记录。
- 不实现跨设备同步最近活动记录。
- 不改变浏览器默认搜索引擎、主页或地址栏搜索行为。
- 不引入 React/Vue 等前端框架，继续使用当前原生 HTML/CSS/JS 架构。
- 不在本次变更中重新设计 popup 或 side panel。

## Decisions

### 决策：新标签页采用分类导航优先布局

桌面布局改为：

- 顶部：TabTrail 标题、标签搜索、当前窗口/全部窗口范围切换、摘要数量。
- 分类标题区：`全部`、手动分类、网站域名分类、`最近关闭` 以网格按钮展示，每个分类显示名称和数量。
- 内容区：只展示当前选中分类的标签列表，减少纵向信息噪音。
- 辅助区：最近活跃和最近关闭可以作为次级区域保留，但不能抢占主分类导航。
- 操作区：归类已选、关闭已选、关闭当前分类、应用排序等动作贴近当前分类内容。

替代方案是继续保留三栏，只优化视觉样式。该方案改动小，但不能解决“分类标题一行几个、先扫分类”的核心问题。

### 决策：分类标题既是导航，也是拖拽目标

分类标题按钮承担两个用途：

- 点击：切换当前分类。
- 拖拽悬停/释放：把打开标签归入该分类。

为了避免误操作，拖拽目标只对可写手动分类开放。域名自动分类不作为写入目标，因为域名分类由 URL 自动生成；`最近关闭` 是只读系统分类，也不作为写入目标。

替代方案是允许拖到任意分类标题。该方案看似直接，但拖到域名分类会产生“手动分类”和“网站分组”语义冲突，不利于用户理解。

### 决策：关闭分类只关闭当前打开标签

关闭分类动作只处理普通分类中的当前打开标签：

- 手动分类：关闭该分类中当前打开的标签。
- 网站域名分类：关闭该域名分组中当前打开的标签。
- `全部`：不提供关闭分类按钮，避免一次关闭整个当前范围。
- `最近关闭`：不提供关闭分类按钮，因为它不是打开标签集合。

关闭 1 个或 2 个标签也必须确认，因为“关闭分类”是批量语义，风险高于单个勾选关闭。确认文案必须包含分类名和数量。

### 决策：新建分类采用轻量表单

仪表盘提供一个轻量分类输入区：输入分类名后，可对当前选中标签执行“归类已选”。如果没有选中标签，只创建空分类会带来状态持久化和空分类维护问题，因此第一版不创建空分类。用户需要先选中标签再创建或指定分类。

### 决策：移除新标签页 side panel 入口

新标签页本身是默认整理入口，继续展示“打开侧边栏”会让主流程不明确。本变更移除该按钮和 `openDashboardSidePanel` 相关调用。侧边栏仍可从 popup 或浏览器扩展入口打开。

### 决策：版本与 README 放在最后提交

版本升级和 README 必须在功能完成、测试通过后处理。版本号从 `0.1.0` 升到 `0.2.0`，并保持 `manifest.json` 与 `package.json` 一致。README 使用中文，面向用户说明安装、核心功能、权限用途、隐私边界、浏览器限制和使用流程。

## Risks / Trade-offs

- 风险：分类标题同时支持点击和拖拽，交互含义变多。缓解：只在拖拽时显示可投放状态，最近关闭和域名分类明确禁用投放。
- 风险：大量分类导致标题区拥挤。缓解：使用响应式网格和稳定最小宽度，窄屏下自动换行。
- 风险：关闭分类误伤多个标签。缓解：所有关闭分类动作都必须确认，并在确认文案中展示分类名和数量。
- 风险：手动分类与域名分类重复展示同一标签。缓解：当前分类内容区只展示选中分类，不同时展开所有分类。
- 风险：移除新标签页 side panel 入口后，用户找不到侧边栏。缓解：popup 保留侧边栏入口，README 说明入口差异。

## Migration Plan

不需要数据迁移。已有 `manualCategories`、`pinnedKeys`、最近活动记录和新标签页覆盖配置继续使用。实现完成后运行测试和构建，再升级版本并补 README。

若用户不喜欢新标签页覆盖，可通过禁用扩展恢复浏览器默认新标签页；本次不新增运行时开关。

## Open Questions

无。当前决策采用用户最新反馈：分类标题横向网格、移除打开侧边栏、支持拖拽归类、新建分类、关闭分类，并最后升级版本和 README。
