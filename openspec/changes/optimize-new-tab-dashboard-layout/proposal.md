## Why

当前新标签页仪表盘采用三栏工作台布局，分类以纵向列表堆叠，用户打开很多标签时需要滚动查找，不符合“先扫分类、再看内容”的使用习惯。用户希望仪表盘更接近资讯聚合站的分类导航体验：一行展示多个分类标题，快速切换、拖拽归类，并能直接关闭某个分类。

同时，新标签页已经成为主要入口，不应再把“打开侧边栏”作为核心动作。侧边栏可以保留为独立入口，但仪表盘本身要能完成分类、新建分类、批量关闭和关闭分类等主要整理操作。

## What Changes

- 重构新标签页仪表盘排版：从三栏卡片式布局改为“顶部搜索与范围控制 + 分类标题网格 + 当前分类内容区 + 辅助最近记录”的布局。
- 分类标题区一行展示多个分类标题，显示分类名、数量和状态，支持点击切换当前分类。
- 支持在仪表盘中新建手动分类，并将选中标签归入该分类。
- 支持把打开标签拖拽到分类标题上完成归类。
- 支持直接关闭一个普通分类中的全部当前打开标签，关闭前必须确认。
- 保留“最近关闭”为只读系统分类：可查看和重新打开，但不能拖入、不能编辑、不能被关闭分类操作处理。
- 从新标签页仪表盘移除“打开侧边栏”按钮和相关交互说明。
- 保持不修改浏览器默认搜索引擎、主页或地址栏搜索行为。
- 最后将插件版本从 `0.1.0` 升一级，并补充项目 `README.md`，说明功能、安装、权限、限制和使用方式。

## Capabilities

### New Capabilities

- `new-tab-category-dashboard`: 定义新标签页仪表盘的分类导航布局、当前分类内容区、拖拽归类、新建分类和关闭分类能力。
- `extension-release-readiness`: 定义插件版本升级和 README 用户文档要求。

### Modified Capabilities

- `new-tab-dashboard`: 移除新标签页中的 side panel 入口要求，并把仪表盘定位从三栏工作台调整为分类优先的默认整理页。
- `tab-organization-panel`: 扩展手动分类能力，使新标签页仪表盘也能创建分类、拖拽归类和关闭分类，同时继续保护最近关闭系统分类。

## Impact

- 影响 `src/newtab/newtab.html`、`src/newtab/newtab.css`、`src/newtab/newtab.js` 和 `src/newtab/newtab-model.js`。
- 影响共享整理模型：可能需要在 `src/sidepanel/sidepanel-model.js` 中增加可复用的分类关闭、拖拽归类或分类摘要函数。
- 影响测试：需要补充新标签页 UI、模型、拖拽归类、关闭分类、只读最近关闭保护、版本和 README 测试。
- 影响 manifest/package 元数据：最后需要同步升级 `manifest.json` 和 `package.json` 版本号。
- 不引入后端服务，不读取网页正文，不上传最近记录。
- 不改变浏览器默认搜索引擎、主页或地址栏搜索行为。
