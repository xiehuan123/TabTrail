## 背景与动机

当前 TabTrail 已具备最近活跃、最近关闭、侧边栏整理等基础能力，但用户需要手动打开侧边栏，界面信息层级不够清晰，最近记录缺少时间线感，导致第一次使用和日常高频使用都不够顺手。

本变更目标是提升插件的可用性和上手效率：让用户更容易打开/保持侧边栏、快速理解最近标签状态、通过更清晰的侧边栏布局完成整理，并提供隐藏浏览器顶部标签栏的可行替代能力或明确边界。

在调研 Chromium 扩展能力后，本变更同时新增“TabTrail 新标签页仪表盘”：用扩展的 `chrome_url_overrides.newtab` 替换浏览器默认新标签页，让用户每次新建标签时直接进入标签整理工作台，而不是必须手动打开 side panel。

## 变更内容

- 改造 popup，使其成为“快速入口中心”，提供打开/切换侧边栏、最近活跃、最近关闭和基础状态提示。
- 在最近活跃和最近关闭列表中显示相对时间或绝对时间，帮助用户判断记录新旧。
- 优化 side panel 信息架构，使用更清晰的顶部工具区、分区摘要、标签列表和操作区。
- 降低手动打开侧边栏成本：提供 popup 一键打开侧边栏，并在支持的浏览器中配置点击扩展图标打开侧边栏的选项。
- 新增新标签页仪表盘，让用户新建标签时默认看到 TabTrail 的整理视图、最近活跃、最近关闭和快速操作。
- 提供“专注整理模式”能力：通过收纳/移动标签到 TabTrail 管理视图、批量关闭或保留可恢复记录，减少顶部标签栏拥挤。
- 明确浏览器顶部标签栏隐藏能力的技术边界：普通扩展不能直接隐藏浏览器原生标签栏；只能提供替代性的收纳、侧边栏管理、窗口整理或说明用户需要浏览器/主题级设置。
- 采用 UI/UX Pro Max 作为界面验收标准，约束信息架构、视觉层级、触控尺寸、可访问性、状态反馈和响应式表现。
- 改善空状态、错误状态和首次使用引导，使用户知道下一步该做什么。
- 保持隐私边界：不读取网页正文、不上传最近记录、不改变最近关闭系统分类的只读原则。

## 能力范围

### 新增能力

- `extension-entry-usability`: 改善 popup 和侧边栏入口，让用户能更快打开、切换和理解 TabTrail。
- `recent-list-time-context`: 在最近活跃和最近关闭中展示时间信息，并定义时间格式和刷新规则。
- `side-panel-layout-refresh`: 重构 side panel 布局，使标签整理、搜索、分类、排序、批量操作更清晰。
- `tab-strip-focus-mode`: 提供减轻浏览器顶部标签栏拥挤的替代方案，并明确不能直接隐藏原生标签栏的边界。
- `new-tab-dashboard`: 将浏览器新标签页替换为 TabTrail 仪表盘，承载更完整的标签整理工作台。
- `ui-ux-pro-max-quality`: 定义本次 UI 改版必须满足的 UI/UX Pro Max 质量标准。

### 修改能力

无。

## 影响范围

- 影响 popup UI、side panel UI、共享时间格式化工具、侧边栏打开逻辑和安装/使用说明文档。
- 影响 manifest 配置：可能新增 `chrome_url_overrides.newtab`，并调整 side panel 打开行为。
- 影响新增页面：需要新增 `src/newtab` 页面及其样式、交互和测试。
- 不引入后端服务，不上传浏览数据，不读取网页正文。
- 不承诺直接隐藏浏览器原生顶部标签栏；该能力受浏览器扩展 API 限制。
- 不修改用户默认搜索引擎；新标签页中的搜索或跳转入口必须清晰说明其作用范围。
