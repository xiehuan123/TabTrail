## 背景

TabTrail 当前 MVP 已实现 popup、side panel、最近活跃、最近关闭、手动分类、搜索、排序和批量关闭等基础能力。用户反馈集中在实际可用性：入口不够顺手、侧边栏需要手动打开、UI 排版不够清晰、最近记录缺少时间信息，以及浏览器顶部标签栏仍然拥挤。

普通 Chromium 扩展不能直接隐藏浏览器原生顶部标签栏，这是浏览器 UI 层能力，不是扩展 API 能力。本设计会提供替代方案：让侧边栏更容易打开、更适合长期停留，并提供“专注整理模式”帮助用户把真实标签栏压力降下来。

Chromium 扩展可以通过 `chrome_url_overrides.newtab` 替换新标签页。因此本变更把“新标签页仪表盘”作为主要入口之一：用户新建标签时直接看到 TabTrail 工作台，用更大的空间展示最近活跃、最近关闭、当前标签分组和整理操作。该能力不会改变默认搜索引擎，也不能覆盖无痕窗口的新标签页。

## 目标 / 非目标

**目标：**

- 让 popup 成为快速入口中心，能一键打开 side panel，并展示最近状态摘要。
- 在最近活跃和最近关闭记录中显示时间信息。
- 重构 side panel 为更易扫读的工作台布局：顶部状态区、工具区、最近区、分组标签区、操作区。
- 支持浏览器能力允许时，通过扩展图标或 popup 按钮打开 side panel。
- 新增 TabTrail 新标签页仪表盘，作为用户打开新标签时默认看到的整理页面。
- 提供“专注整理模式”，通过批量收纳、关闭后可恢复、当前窗口整理等方式减少顶部标签栏拥挤。
- 按 UI/UX Pro Max 标准改造界面，保证信息层级清楚、触控/键盘可用、状态反馈完整、文本不重叠。
- 在 UI 和文档中明确说明：TabTrail 不能直接隐藏浏览器原生标签栏，只能提供替代整理方式。

**非目标：**

- 不实现浏览器原生顶部标签栏的强制隐藏。
- 不实现动态开关新标签页覆盖；扩展启用时新标签页覆盖由 manifest 静态声明决定。
- 不修改浏览器默认搜索引擎、主页或地址栏搜索行为。
- 不实现云端同步最近记录。
- 不读取网页正文内容。
- 不引入复杂前端框架，优先沿用当前原生 HTML/CSS/JS 架构。
- 不改变“最近关闭”只读系统分类原则。

## 关键决策

### 决策：popup 作为入口中心，而不是完整管理界面

Popup 空间有限，适合做快速入口和最近状态摘要。完整整理仍放在 side panel。Popup 应提供“打开侧边栏”、“最近活跃”、“最近关闭”和简短状态提示，不增加批量分类、拖拽排序等重操作。

替代方案是在 popup 内做完整管理，但会导致布局拥挤，且与既有侧边栏定位重复。

### 决策：side panel 改成工作台式布局

Side panel 需要服务“长时间整理很多标签”的场景，应从简单列表变成分层工作台：

- 顶部：当前范围、标签数量、最近活跃/最近关闭摘要。
- 工具区：搜索、范围切换、分类输入、批量操作。
- 主体：最近活跃、按网站分组、手动分类、最近关闭。
- 底部或固定工具：应用到标签栏、清空最近关闭等危险/确认操作。

这样用户能先看到状态，再决定操作，而不是被一长串标签淹没。

### 决策：新标签页仪表盘作为默认工作台

新标签页比 popup 和 side panel 有更大空间，适合作为“默认工作台”。第一版使用三栏响应式结构：

- 顶部命令区：搜索当前标签、打开 side panel、切换当前窗口/全部窗口、展示打开标签数量。
- 左侧最近区：最近活跃和最近关闭，必须显示时间，并保留最近关闭只读分类原则。
- 中央整理区：按网站/手动分类展示当前打开标签，支持点击激活、搜索过滤、选择和批量整理。
- 右侧操作区：应用排序、批量关闭、专注整理说明与确认入口。

窄屏或小窗口下改为单列分区，最近区和操作区折叠到主列表上下方，避免横向滚动和文本重叠。

### 决策：UI/UX Pro Max 标准落到可验收规则

本次 UI 不做营销页或装饰性首页，而是做高频生产力工具界面。视觉方向为“安静、清晰、密集但不拥挤”：使用克制的中性色底、明确的语义色、8px 以内圆角、统一 SVG 图标、稳定控件尺寸和清晰焦点态。

关键验收规则：

- 所有可点击控件有明确标签或 `aria-label`，键盘焦点可见。
- 主要触控目标至少 44px 高，紧凑控件必须保留足够点击区域。
- 正文对比度达到 4.5:1，状态颜色不得作为唯一信息来源。
- popup、side panel、新标签页在目标宽度下不得出现横向滚动、文本溢出或元素重叠。
- 空状态、加载状态、错误状态和确认状态必须可见，并给出直接动作。
- 动效只用于状态变化反馈，时长控制在 150-300ms，并尊重 `prefers-reduced-motion`。

### 决策：时间显示使用相对时间优先

最近列表默认显示“刚刚 / 5 分钟前 / 今天 14:30 / 昨天 09:10”等人类可读时间。相对时间更适合快速判断“是不是刚才那个标签”。需要保留完整时间作为 tooltip 或辅助文本，以便精确判断。

### 决策：隐藏顶部标签栏采用替代方案表达

由于扩展不能直接隐藏浏览器原生标签栏，产品不承诺“隐藏原生标签栏”。改为提供“专注整理模式”：

- 引导用户打开 side panel 并保持使用。
- 提供按窗口整理和批量关闭。
- 关闭后进入最近关闭，可恢复。
- 第一版不做独立离线收纳库；先复用最近关闭和现有偏好数据，避免用户误以为标签被云端保存。

### 决策：入口便利优先使用官方 API

优先使用 `chrome.sidePanel.open` 和 `chrome.sidePanel.setPanelBehavior` 等官方能力。若某浏览器不支持某个入口能力，UI 必须降级为说明和备用按钮，而不是静默失败。

## 风险 / 取舍

- 风险：用户期望真正隐藏顶部标签栏。缓解：UI 和文档明确说明浏览器限制，并提供专注整理替代方案。
- 风险：side panel 信息过多导致更复杂。缓解：分区折叠、摘要优先、主要操作固定在顶部。
- 风险：相对时间会过期。缓解：popup/side panel 打开时计算一次，面板停留时按分钟刷新。
- 风险：新标签页覆盖会影响用户已有搜索习惯。缓解：页面顶部保留清晰的地址栏/搜索引擎边界说明和快速打开常用搜索入口，但不修改默认搜索引擎。
- 风险：不同 Chromium 浏览器 side panel API 行为不完全一致。缓解：能力检测，Chrome 优先，Edge 做基础兼容验证。
- 风险：新增 UI 改动破坏现有功能。缓解：为时间格式、入口状态、side panel 状态模型和关键交互加测试。

## 迁移计划

该变更不需要数据迁移。已有最近记录和偏好数据继续使用。实现时先增加共享时间工具、UI token 和入口能力检测，再重构 popup 与 side panel UI，最后新增 newtab 仪表盘。

## 已确认边界

- 专注整理模式第一版提供说明、批量关闭确认、恢复入口和窗口整理，不做独立离线收纳库。
- 最近时间使用中文 24 小时制表达，打开页面时立即计算，页面停留时按分钟刷新。
- 新标签页覆盖作为本变更的一部分实施；用户若要恢复浏览器默认新标签页，需要禁用扩展或后续使用单独构建/设置方案。
